# 論文比較分析スキル

複数の論文のitemKeyまたはキーワードと引数が `$ARGUMENTS` で渡される。

## 引数のパース

`$ARGUMENTS` を以下のルールでパースせよ：

- itemKey形式（英数字ちょうど8文字）のトークンを複数受け取る → `<itemKeys>` のリスト
- または、`--save` / `--focus` / `--limit` 以外の連続トークン → `<query>`（キーワード検索）
- `--save` → 比較結果をZoteroへノート保存（各論文にそれぞれ保存）
- `--focus <aspect>` → 比較の焦点（例: `method`, `data`, `evaluation`）。省略時は全般比較
- `--limit <N>` → キーワード検索時の比較対象数（デフォルト: 検索結果から選択）
- itemKeyもqueryも見つからない場合はエラーメッセージを出して終了
- itemKeyが1つしか指定されていない場合は「比較には2つ以上の論文が必要です」と表示して終了

## 手順

### Step 1a: itemKeyの解決

**複数のitemKeyが直接指定された場合：**

そのまま `<itemKeys>` リストとして Step 1b へ進む。

**queryがキーワードの場合：**

Bashツールで以下を実行：

```
cd ~/zotero-cli && ./zotero-cli search <query>
```

検索結果に応じて分岐：

- **0件** → 「検索結果が見つかりませんでした: `<query>`」と表示して終了
- **1件** → 「比較には2つ以上の論文が必要です」と表示して終了
- **複数件** →
  - `--limit <N>` 指定あり → 上位N件を自動採用
  - `--limit` なし → 候補一覧を番号付きで表示し、AskUserQuestion ツールでユーザーに比較する論文の番号を複数選択してもらう（例: `1,3,5` またはスペース区切り）

候補一覧の表示形式：

```
検索結果: <N>件見つかりました。比較する論文の番号をカンマ区切りで入力してください（例: 1,3,5）。

  1. [FQVL7ZHM] Investigating Environmental, Social, and Governance... (Angioni et al.)
  2. [99NU4NKK] An automated information extraction system... (Mohsin et al., 2024)
  ...
```

### Step 1b: 各論文の情報取得

選択された各itemKeyについて、Bashツールで以下を実行（可能な限り並列で）：

```
cd ~/zotero-cli && ./zotero-cli context <itemKey> --json
```

いずれかのコマンドが失敗した場合は、失敗したitemKeyを報告し、成功した論文のみで比較を続行する。成功した論文が2本未満の場合はエラーを表示して終了。

### Step 2: 比較分析の生成

取得したJSONから各論文の情報を読み取り、以下の構造で**日本語で**比較分析を生成する。

`--focus` が指定されている場合は、その側面を重点的に比較する。

```
## ⚖️ 論文比較分析

### 比較対象

| # | 論文 | 著者 | 年 |
|---|------|------|----|
| 1 | <タイトル1> | <著者1> | <年1> |
| 2 | <タイトル2> | <著者2> | <年2> |
| ... | ... | ... | ... |

### 比較表

| 観点 | 論文1 | 論文2 | ... |
|------|-------|-------|-----|
| **研究目的** | ... | ... | ... |
| **手法** | ... | ... | ... |
| **データセット** | ... | ... | ... |
| **評価指標** | ... | ... | ... |
| **主要結果** | ... | ... | ... |
| **限界** | ... | ... | ... |

### 総合分析

#### 共通点
- （論文間の共通するアプローチ・知見）

#### 相違点
- （重要な違い・対照的なアプローチ）

#### 研究ギャップ
- （これらの論文群が埋めきれていない領域）

#### 示唆
- （比較から得られる洞察・自身の研究への示唆）
```

### Step 3: ノート保存（`--save` 指定時のみ）

`--save` が指定されている場合のみ実行：

1. 生成した比較分析をtempfileに書き出す
2. 比較対象の**各itemKey**に対して、Bashツールで以下を実行：

```
cd ~/zotero-cli && ./zotero-cli add-note <itemKey> --body-file <tempfile> --tags "ai-compare"
```

3. tempfileを削除
4. 保存完了メッセージを表示（保存先の各itemKeyを列挙）

`--save` が指定されていない場合は、比較分析を表示するだけで終了。
