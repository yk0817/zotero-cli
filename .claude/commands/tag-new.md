# 新規タグ作成スキル

論文の itemKey と、付けたい新規タグが `$ARGUMENTS` で渡される。

このスキルは **「Zotero のタグ語彙を意図的に拡張する唯一の入口」** である。
通常のタグ付け（`tag` スキル）は既存タグだけに閉じる（クローズド語彙）が、こちらは
**新しいタグを作ることを明示的な目的**とする。それゆえ、新タグを安易に増やして
語彙を汚さないよう、付与前に既存タグとの重複・類似チェックを必ず行う。

## 引数のパース

`$ARGUMENTS` を以下のルールでパースせよ：

- 最初のトークン（`--tag` / `--dry-run` 以外の連続トークン）→ `<query>`（必須・対象論文）
- `--tag "<新タグ>"` → 作って付けたい新規タグ（**必須・複数指定可**）
- `--dry-run` → 付与せず、付ける予定のタグと結果のタグ集合をプレビューするだけ
- `<query>` または `--tag` が見つからない場合はエラーメッセージを出して終了

### queryの判定

`<query>` がitemKey形式（英数字ちょうど8文字、例: `FQVL7ZHM`）かどうかを判定する：

- **itemKey形式に一致** → そのまま `<itemKey>` として使用
- **一致しない** → タイトル/キーワード検索として扱う（次の Step 1a で検索）

## 手順

### Step 1a: itemKeyの解決

**queryがitemKey形式の場合：** そのまま `<itemKey>` として Step 2 へ進む。

**queryがキーワードの場合：**

```
cd ~/zotero-cli && ./zotero-cli search <query>
```

- **0件** → 「検索結果が見つかりませんでした: `<query>`」と表示して終了
- **1件** → その itemKey を自動採用し、Step 2 へ
- **複数件** → 候補一覧を番号付きで表示し、AskUserQuestion でユーザーに選択してもらう

### Step 2: 既存タグとの重複・類似チェック（最重要）

Bashツールで既存タグ一覧を取得する：

```
cd ~/zotero-cli && ./zotero-cli tags --output json
```

`--tag` で指定された各新タグについて、既存タグ一覧と突き合わせる：

- **完全一致がある**（大文字小文字の違いだけを含む）→ それは新規ではない。
  新規作成は中止し、「既存タグ `<既存タグ>` を使ってください（`tag` スキル）」と
  案内して終了する。
- **類似タグがある**（表記ゆれ・単複・日英・略語など。例: `LLM` と `large-language-model`、
  `知識グラフ` と `knowledge-graph`）→ **警告を出して再考を促す**。
  類似候補を提示し、AskUserQuestion で「新タグを作る / 既存タグを使う / 中止」を選ばせる。
- **重複も類似もない** → 新規タグとして妥当。Step 3 へ進む。

### Step 3: 新規タグの付与

確定した新規タグを付与する。

**`--dry-run` 指定時：**

```
cd ~/zotero-cli && ./zotero-cli tag <itemKey> --add "<新タグ>" --dry-run
```

結果のタグ集合をプレビュー表示して終了（書き込みなし）。

**通常時：**

```
cd ~/zotero-cli && ./zotero-cli tag <itemKey> --add "<新タグ>"
```

複数の新タグは `--add` を繰り返して指定する。

### Step 4: 完了報告

付けた新規タグと、結果のタグ集合を表示する。新タグを1つでも作った場合は、
「ライブラリの語彙を拡張した（新タグ: …）」と明示し、以後そのタグは `tag`
スキルのクローズド語彙にも含まれるようになることを一言添える。
