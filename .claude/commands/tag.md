# クローズド語彙タグ付けスキル

論文の itemKey と引数が `$ARGUMENTS` で渡される。

このスキルの肝は **「Zotero に新しいタグを乱立させない」** こと。付けるタグは
**ライブラリにすでに存在するタグだけ**に限定する（クローズド語彙）。既存にない
概念を新タグとして付けたくなったら、それは別スキル `tag-new` の仕事であり、この
スキルでは**絶対に作らない**。

## 引数のパース

`$ARGUMENTS` を以下のルールでパースせよ：

- 最初のトークン（`--dry-run` / `--max` 以外の連続トークン）→ `<query>`（必須）
- `--dry-run` → 付与せず、付ける予定のタグと結果のタグ集合をプレビューするだけ
- `--max <N>` → このスキルが新たに付与するタグ数の上限（省略時は `5`）
- `<query>` が見つからない場合はエラーメッセージを出して終了

### queryの判定

`<query>` がitemKey形式（英数字ちょうど8文字、例: `FQVL7ZHM`）かどうかを判定する：

- **itemKey形式に一致** → そのまま `<itemKey>` として使用
- **一致しない** → タイトル/キーワード検索として扱う（次の Step 1a で検索）

## 手順

### Step 1a: itemKeyの解決

**queryがitemKey形式の場合：** そのまま `<itemKey>` として Step 2 へ進む。

**queryがキーワードの場合：**

Bashツールで以下を実行：

```
cd ~/zotero-cli && ./zotero-cli search <query>
```

検索結果に応じて分岐：

- **0件** → 「検索結果が見つかりませんでした: `<query>`」と表示して終了
- **1件** → その itemKey を自動的に採用し、Step 2 へ進む
- **複数件** → 候補一覧を番号付きで表示し、AskUserQuestion ツールでユーザーに番号を選択してもらう。選択された itemKey で Step 2 へ進む

候補一覧の表示形式：

```
検索結果: <N>件見つかりました。番号を入力して選択してください。

  1. [FQVL7ZHM] Investigating Environmental, Social, and Governance... (Angioni et al.)
  2. [99NU4NKK] An automated information extraction system... (Mohsin et al., 2024)
  ...
```

### Step 2: 論文内容の把握

Bashツールで以下を実行し、論文のメタdata・ノート（あれば）を取得する：

```
cd ~/zotero-cli && ./zotero-cli context <itemKey> --json --with-notes
```

タイトル・アブストラクトだけで分類が不十分なら、本文も取得する：

```
cd ~/zotero-cli && ./zotero-cli fulltext <itemKey> --max-chars 8000
```

コマンドが失敗した場合はエラーを表示して終了。

### Step 3: 既存タグ一覧（クローズド語彙）の取得

Bashツールで以下を実行し、ライブラリの**既存タグ集合**を取得する：

```
cd ~/zotero-cli && ./zotero-cli tags --output json
```

ここで返ってきたタグだけが、このスキルで付けてよいタグの**全集合**である。
`numItems`（使用回数）も付くので、確立したタグか一度きりのタグかの判断材料にする。

### Step 4: 付与するタグを既存集合の中からのみ選ぶ

論文内容（Step 2）と既存タグ一覧（Step 3）を突き合わせ、内容に合うタグを
**既存タグ集合の中からのみ**選ぶ。

- **集合外のタグは絶対に選ばない・作らない。** 「あったら良さそうな新タグ」を
  ここで付けてはいけない。
- 付与数は `--max <N>`（既定5）を超えない。内容に強く合致する順に上位 N 件を選ぶ。
- すでにその論文に付いているタグは数えない（差分として新たに付くものだけを上限の対象にする）。

### Step 5: 未登録だが付けたい概念の提示（付与はしない）

論文に本当は付けたいのに既存タグ一覧に無い概念があれば、それは
**「候補（未登録・このスキルでは付与しない）」** として列挙して提示するだけにとどめる。

```
## 付与したタグ（既存語彙から選択）
- <既存タグ1>
- <既存タグ2>

## 候補（未登録・付与せず）
新規タグとして登録したい場合は `tag-new` スキルを使ってください。
- <未登録の概念1>
- <未登録の概念2>
```

### Step 6: 付与の実行

選んだ既存タグを付与する。

**`--dry-run` 指定時：**

```
cd ~/zotero-cli && ./zotero-cli tag <itemKey> --add "<タグ1>" --add "<タグ2>" --dry-run
```

結果のタグ集合をプレビュー表示して終了（実際の書き込みはしない）。

**通常時：**

```
cd ~/zotero-cli && ./zotero-cli tag <itemKey> --add "<タグ1>" --add "<タグ2>"
```

付与後、付けたタグと結果のタグ集合、および Step 5 の候補（あれば）を表示して終了。
