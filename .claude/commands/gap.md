# 研究ギャップ抽出スキル

引数が `$ARGUMENTS` で渡される。

複数論文（コレクション / タグ / 複数 itemKey）を横断して、**未解決の研究ギャップ**と**候補リサーチクエスチョン**を列挙する。`survey-table`（文献を表で整理する）・`related-work`（関連研究の散文を書く）とは別軸で、「次に何を問うべきか」を出すのが目的。

## 引数のパース

`$ARGUMENTS` を以下のルールでパースせよ：

- `--tag <tag>` → 指定タグの論文を対象にする
- `--collection <collectionKey>` → 指定コレクションの論文を対象にする
- `--keys "<key1>,<key2>,..."` → カンマ区切りのitemKeyリストで対象を指定
- `--limit <N>` → 対象論文数の上限（デフォルト: 20）
- `--save` → 生成したギャップ分析をZoteroにノート保存
- 上記のいずれの指定もない場合は、`$ARGUMENTS` 全体をキーワード検索として扱う

対象指定が見つからない場合はエラーメッセージを出して終了。

## 手順

### Step 1: 対象論文群の取得

指定方法に応じてBashツールで以下を実行（`survey-table` と同じ入力収集方式）：

**タグ指定の場合：**

```
cd ~/zotero-cli && ./zotero-cli export --tag <tag> --format json --limit <limit>
```

**コレクション指定の場合：**

```
cd ~/zotero-cli && ./zotero-cli export --collection <collectionKey> --format json --limit <limit>
```

**itemKey指定の場合：**

```
cd ~/zotero-cli && ./zotero-cli export --keys "<key1>,<key2>,..." --format json
```

**キーワード検索の場合：**

```
cd ~/zotero-cli && ./zotero-cli search <query>
```

検索結果に応じて分岐：

- **0件** → 「検索結果が見つかりませんでした: `<query>`」と表示して終了
- **複数件** → 候補一覧を表示し、AskUserQuestion ツールでユーザーに対象の番号を複数選択してもらう

選択後、各itemKeyについて `context --json` で詳細を取得する。

コマンドが失敗した場合はエラーを表示して終了。結果が0件の場合は「対象の論文が見つかりませんでした」と表示して終了。ギャップ分析には**最低2本**の論文が望ましい。1本のみの場合は「横断分析には複数論文が必要です」と注意を添えた上で実行してよい。

### Step 2: 各論文の要点を内部把握

取得したJSON（および必要に応じて `context --json`）から、各論文について以下を内部的に把握する（この段階では出力しない）：

- 研究目的・扱う問題
- 手法・データ
- 主要な貢献と、明示的に述べられている限界（Limitations / Future Work）
- どこまで解けて、何が未解決として残っているか

推測で補った点は後段の出力で「推測」と明示できるよう区別しておく。捏造はしない。

### Step 3: ギャップ分析の出力生成

取得した論文情報をもとに、**日本語で**以下の構成で出力する。各主張は対象論文に紐付け、論文から読み取れない点は「推測」と明示する。

```
## 🔍 研究ギャップ分析

**対象:** <タグ名/コレクション名/検索キーワード>
**論文数:** <N>本

### 対象論文一覧

1. Author et al. (Year) — Title
2. Author et al. (Year) — Title
... （N件）

### 共通テーマ / 現状の到達点

<この論文群が扱う領域で、何が分かっているか・どこまで到達しているかを2〜4文で要約。>

### 研究ギャップ

- **<ギャップ1の見出し>** — なぜ未解決か。どの論文がどこまでやったか（例: \cite なしで「Author (Year) は X を示したが Y は未検証」）。
- **<ギャップ2の見出し>** — 同上。
- **<ギャップ3の見出し>** — 同上。
（必要に応じて増減。各ギャップに必ず「なぜ未解決か / どの論文がどこまでやったか」を添える）

### 候補リサーチクエスチョン

各ギャップに対応する、次に問うべき問いを 3〜5 個。各々に簡単な狙い。

1. **RQ1:** <問い> — 狙い: <一言>
2. **RQ2:** <問い> — 狙い: <一言>
3. **RQ3:** <問い> — 狙い: <一言>
（最大5個まで）

### アプローチの示唆

各 RQ に対し考えられる手法の方向性を一言で。

- **RQ1:** <手法の方向性（例: ○○データセットで△△を適用、など）>
- **RQ2:** <手法の方向性>
- **RQ3:** <手法の方向性>
```

注意：

- 根拠は必ず対象論文に紐付ける。論文に書かれていない推論は「推測」と明示する。
- 存在しない論文・数値・結果を捏造しない。
- ギャップと RQ は対応づける（どのギャップに答える RQ かを読者が辿れるようにする）。

### Step 4: ノート保存（`--save` 指定時のみ）

`--save` が指定されている場合のみ実行：

1. 生成したギャップ分析をtempfileに書き出す。
2. **保存先 itemKey の決定**（複数論文が対象なので注意）：
   - itemKey が1つに定まる場合（`--keys` で1件、または検索で1件選択）は、その itemKey に保存する。
   - 対象が複数論文の場合、保存先が一意でない。AskUserQuestion ツールで「どの論文のノートとして保存するか」をユーザーに確認する（対象論文一覧を選択肢として提示）。ユーザーが応答しない・判断を委ねる場合は、対象集合の**代表として最初のitemKey**に保存する。
3. 決定した itemKey に対して、Bashツールで以下を実行：

```
cd ~/zotero-cli && ./zotero-cli add-note <targetItemKey> --body-file <tempfile> --tags "ai-gap"
```

4. tempfileを削除。
5. 保存完了メッセージ（保存先の論文タイトル・itemKey を明記）を表示。

`--save` が指定されていない場合は、ギャップ分析を表示するだけで終了。
