# 再現性メソッド抽出スキル

論文のitemKeyと引数が `$ARGUMENTS` で渡される。

このスキルは「読み物としての要約」(summarize) とは別軸で、**実験を再現するのに必要な事実だけ**を本文から構造化抽出する。所感・評価・物語は含めず、表に落とせる事実（データセット名・規模・分割・ハイパーパラメータ・評価指標・代表数値・計算資源・コード公開URL等）を集める。

## 引数のパース

`$ARGUMENTS` を以下のルールでパースせよ：

- 最初のトークン（または `--save` 以外の連続トークン）→ `<query>`（必須）
- `--save` → 抽出後にZoteroへノート保存
- `<query>` が見つからない場合はエラーメッセージを出して終了

### queryの判定

`<query>` がitemKey形式（英数字ちょうど8文字、例: `FQVL7ZHM`）かどうかを判定する：

- **itemKey形式に一致** → そのまま `<itemKey>` として使用
- **一致しない** → タイトル/キーワード検索として扱う（次のStep 1aで検索）

## 手順

### Step 1a: itemKeyの解決

**queryがitemKey形式の場合：**

そのまま `<itemKey>` として Step 1b へ進む。

**queryがキーワードの場合：**

Bashツールで以下を実行：

```
cd ~/zotero-cli && ./zotero-cli search <query>
```

検索結果に応じて分岐：

- **0件** → 「検索結果が見つかりませんでした: `<query>`」と表示して終了
- **1件** → その itemKey を自動的に採用し、Step 1b へ進む
- **複数件** → 候補一覧を番号付きで表示し、AskUserQuestion ツールでユーザーに番号を選択してもらう。選択された itemKey で Step 1b へ進む

候補一覧の表示形式：

```
検索結果: <N>件見つかりました。番号を入力して選択してください。

  1. [FQVL7ZHM] Investigating Environmental, Social, and Governance... (Angioni et al.)
  2. [99NU4NKK] An automated information extraction system... (Mohsin et al., 2024)
  ...
```

### Step 1b: 論文情報（メタ情報）の取得

Bashツールで以下を実行：

```
cd ~/zotero-cli && ./zotero-cli context <itemKey> --json --with-notes
```

コマンドが失敗した場合はエラーを表示して終了。ここで取得するのはタイトル・著者・年・出版先・アブストラクト等のメタ情報。

### Step 1c: 本文の取得

再現性メソッドの抽出には**本文**が必要。Bashツールで以下を実行：

```
cd ~/zotero-cli && ./zotero-cli fulltext <itemKey>
```

- **本文が取得できた場合** → 本文を一次情報として Step 2 の抽出に使う。
- **本文が取得できなかった場合**（添付PDFが無い / Zotero未同期 / 全文索引が空）→ Step 1b で取得した context のアブストラクト等から取れる範囲だけで抽出し、出力の冒頭に次の注記を明示する：

  ```
  > ⚠️ 本文が取得できなかったため（添付PDF無し/未同期の可能性）、以下はアブストラクト等から取れる範囲の抽出です。多くの項目は推定または不明です。
  ```

本文が長い場合は `--max-chars` で切らず、まず全体を取得して手法・実験・評価セクションを優先して読む。

### Step 2: 再現性メソッド表の生成

取得した本文（取れなければメタ情報）から、**日本語で**以下の構造で抽出する。**捏造はしない** — 本文に書かれていない項目は必ず「不明 / N/A」と明示する。各項目は可能な限り本文の記述に基づく事実のみを書く。

```
## 🔬 再現性メソッド抽出: <タイトル>

**著者:** <著者リスト>
**出版年:** <年>
**出版先:** <ジャーナル/会議>

| 項目 | 内容 |
|------|------|
| **タスク/問題設定** | （何を入力に何を出力する問題か。分類/生成/予測/検索など） |
| **データセット** | （名前 / 規模（件数・トークン数等） / 分割（train/val/test） / 前処理。複数あれば箇条書き） |
| **手法・モデル** | （アーキテクチャ / 主要コンポーネント / ベースとなる既存モデル） |
| **ハイパーパラメータ** | （学習率 / バッチサイズ / エポック数 / 最適化器 / スケジューラ / 正則化 等、判明分。不明な値は明示） |
| **ベースライン/比較手法** | （比較対象として用いた手法・モデル） |
| **評価指標** | （Accuracy / F1 / BLEU / nDCG / RMSE 等、使われた指標） |
| **主要結果** | （代表的な数値。提案手法 vs ベースラインの主要スコアを具体値で） |
| **計算資源** | （GPU種別・枚数 / 学習時間 / メモリ 等、判明分） |
| **再現性メモ** | （コード公開URL / データの入手可否 / ライセンス / 設定が伏せられている不明点・再現の障害） |
```

補足：

- 表に収まりきらない詳細（例: 複数データセットごとの分割や、設定の異なる複数実験）は、表の下にセクションを足して箇条書きで補ってよい。
- 数値は本文の記載をそのまま引く。本文に複数の設定がある場合は代表設定を選び、どの設定かを明記する。
- 「不明」と「該当なし（N/A）」は区別する（前者＝論文に書かれているはずだが読み取れない、後者＝そもそも該当しない）。

### Step 3: ノート保存（`--save` 指定時のみ）

`--save` が指定されている場合のみ実行：

1. 生成した抽出結果をtempfileに書き出す
2. Bashツールで以下を実行：

```
cd ~/zotero-cli && ./zotero-cli add-note <itemKey> --body-file <tempfile> --tags "ai-methods"
```

3. tempfileを削除
4. 保存完了メッセージを表示

`--save` が指定されていない場合は、抽出結果を表示するだけで終了。
