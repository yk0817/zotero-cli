# 論文ディスカッションスキル

論文のitemKeyと引数が `$ARGUMENTS` で渡される。

## 引数のパース

`$ARGUMENTS` を以下のルールでパースせよ：

- 最初のトークン（または `--color` 以外の連続トークン）→ `<query>`（必須）
- `--color <hex>` → 指定色のアノテーションだけを議論の軸にする（例: `--color "#ff0000"`）
- `<query>` が見つからない場合はエラーメッセージを出して終了

### queryの判定

`<query>` がitemKey形式（英数字ちょうど8文字、例: `FQVL7ZHM`）かどうかを判定する：

- **itemKey形式に一致** → そのまま `<itemKey>` として使用
- **一致しない** → タイトル/キーワード検索として扱う（次のStep 1aで検索）

## 手順

### Step 1a: itemKeyの解決

**queryがitemKey形式の場合：**

そのまま `<itemKey>` として Step 1b へ進む。

**queryがキーワードの場合：**

Bashツールで以下を実行：

```
cd ~/zotero-cli && ./zotero-cli search <query>
```

検索結果に応じて分岐：

- **0件** → 「検索結果が見つかりませんでした: `<query>`」と表示して終了
- **1件** → その itemKey を自動的に採用し、Step 1b へ進む
- **複数件** → 候補一覧を番号付きで表示し、AskUserQuestion ツールでユーザーに番号を選択してもらう。選択された itemKey で Step 1b へ進む

### Step 1b: コンテキストの取得

Bashツールで以下を実行（メタデータ + 全文 + アノテーション + ノートが一括で取れる）：

```
cd ~/zotero-cli && ./zotero-cli context <itemKey> --output json
```

`--color` が指定されている場合は、追加で以下を実行し、こちらのアノテーションを議論の軸にする：

```
cd ~/zotero-cli && ./zotero-cli annotations <itemKey> --color "<hex>" --output json
```

コマンドが失敗した場合はエラーを表示して終了。

### Step 2: アノテーションの確認

取得した `annotations` を確認する：

- **0件の場合** → 「この論文には同期済みのアノテーションがありません。Zotero側でハイライトを付けて同期するか、全文ベースで議論を始めることもできます」と案内し、ユーザーの意向を確認する
- **1件以上** → Step 3 へ

アノテーションの型ごとの扱い：

- `highlight` / `underline` — `annotationText` が選択文。`annotationComment` があればユーザーのコメント
- `note` — `annotationComment` が本文（ページ上の注記）
- `ink` / `image` — テキストなし。「手書き（本文を読めない）・p.N」と明示し、全文の該当ページ周辺を読んで内容を推測・案内する

### Step 3: ディスカッションの提示

落合式要約（/summarize）とは**別物**。ユーザーがマークした箇所を軸に、**日本語で**以下の形式で精読の起点を作る：

```
## 💬 精読ディスカッション: <タイトル>

**著者:** <著者リスト> | **出版年:** <年>
**アノテーション:** <N>件（highlight: x, note: y, ink: z）

### マークした箇所を読む

#### 1. [p.<ページ>] "<ハイライト本文（長い場合は要約引用）>"
- **なぜここが重要か:** （論文全体の文脈におけるこの箇所の位置づけ）
- **コメントへの応答:** （annotationCommentがある場合のみ。ユーザーのコメントに対する応答・補足・反論）
- **関連する本文箇所:** （全文から関連する議論・数式・実験結果を引いて接続する）

#### 2. ...（sortIndex順にすべてのアノテーションを扱う）

#### N. [p.<ページ>] 手書きアノテーション（本文を読めない）
- このページの本文では（該当ページの内容を全文から要約）が議論されている。手書きメモの意図に心当たりがあれば教えてほしい。

### 議論の起点

（マークの分布・コメントの傾向から、ユーザーが関心を持っていそうな論点を2-3個挙げ、
 それぞれについて「どう考えるか」を問いかける形で終える）
```

### Step 4: 対話の継続

提示後はユーザーの質問・反応に応じて対話を続ける。全文（fullText）を根拠に、推測と事実を区別しながら答える。本文に書かれていないことは「本文には記載がない」と明示する。
